    前后端分离实践

    　　转眼又到了年底，也是时候做一个年终总结了。关于前后端分离这个概念，在之前就有所了解，但是还未真正付诸实践。终于在今年花了几个月的时间顺利的把前后端分离实践在公司的项目中。

    认识前后端分离

    　　在传统的开发模式下，根据我们以往的认知，可能以为前后端分离的界限就是已浏览器为标准。浏览器中显示的就是前端，而把运行在服务器中的，为前端提供业务逻辑和数据准备的所有代码作为后端，这样来进行一个前后端的划分。但是实际上的前后端分离不是这样，在开发过程中前后端的开发工作的分工就是前后端分离。在开发阶段，前后端工程师约定好数据交互接口，实现并行开发和测试；在运行阶段前后端分离模式需要对web应用进行分离部署，前后端之前使用HTTP或者其他协议进行交互请求。
    我们可以从以下几个方面来理解前后端分离：

    1. 交互形式
    2. 代码组织方式
    3. 开发模式
    4. 数据接口规范流程

    一、交互形式

    　　在前后端分离的开发模式下，前端与后端的交互只需要借助 RESTful api 来进行一个数据的请求就行了，大大地降低了前后端交流的成本。如下图所示：

    二、代码组织方式

    　　在以前的开发模式中，前后端代码可能是耦合在一起的，这样称为前后端未分离情况。公司之前未实现前后端的分离时，前后端代码分别在两个库中进行管理，但是开发模式还不算是前后端分离模式，只能说是半分离模式。前后端分离模式在代码组织形式上有以下两种:

    • 半分离：前后端共用一个代码库，但是代码分别存放在两个工程中。后端不关心或很少 关心前端元素的输出情况，前端不能独立进行开发和测试，项目中缺乏前后端 交互的测试用例。

    • 分离：前后端代码库分离，前端代码中有可以进行Mock测试(通过构造虚拟测试对象以简化测试环境的方法)的伪后端，能支持前端的独立开发和测试。而后端代码中除了功能实现外，还有着详细的测试用例，以保证API的可用性，降低集成风险。

    三、开发模式

    　　在进行开发模式重构之前，前端人员负责页面的编写，逻辑的实现，并且在页面中内嵌一些后端提供的模板变量以及操作逻辑。待后端也开发的差不多之后，前端代码再经过后端人员处理编译成脚本文件，然后一起打包部署到服务器中，最后才进行一个测试工作。测试过程中如果发现问题，则需要：前端返工 => 后端返工 => 代码打包部署 => 测试完成 => 上线。

    　　重构之后，前端开发人员的工作就简化许多了，只需要负责页面结构, 样式以及逻辑的编写。前端页面所需要的数据都是通过 HTTP 请求来获取，在开发之前先约定好接口数据格式，这样后端也只需要专注于 RESTful 接口的开发。流程如下所示：

    　　在开发期间前后端共同商定好数据接口的交互形式和数据格式。然后实现前后端的并行开发，其中前端工程师在开发完成之后可以独自进行mock测试(mock数据方法在我博客中有实现)，而后端也可以使用接口测试平台进行接口自测，然后前后端一起进行功能联调并校验格式，最终进行自动化测试。如下图：

    前后端分离的好处

    　　通过这次的前后端分离实践，我个人感觉有以下几个分离之后有以下几个好处：

    • 提高了开发人员的专注度： 通过前后端分离，前端人员只需要负责前端，后端开发人员只需要负责后端，前后端数据交互通过接口来完成，也不需要套页面以及后端模板变量了，极大地提高了开发人员的专注度，有利于开发人员做到专注专精，开发能力必然会有所提升。

    • 提升了开发效率： 前后端分离以后，可以实现前后端代码的解耦，只要前后端沟通约定好应用所需接口以及接口参数，便可以开始并行开发，无需等待对方的开发工作结束。与此同时，即使需求发生变更，只要接口与数据格式不变，后端开发人员就不需要修改代码，只要前端进行变动即可。

    • 增强代码可维护性： 前后端分离后，应用的代码不再是前后端混合，只有在运行期才会有调用依赖关系。

    前后端分离与服务端渲染

    　　在这次的项目重构中，主要涉及了两种模式的项目。一类是不需要考虑页面 SEO 的， 还有就是需要考虑 SEO 的。因为项目中使用的主要技术是 vue 全家桶，需要考虑 SEO 的话就涉及到了服务端渲染问题。服务端渲染这方面有了解过 nuxt.js 这个工具，在我们的项目实现服务端渲染还是在 node 中间层做了一个处理。比如有一个场景是访问文章页：在访问某个文章的时候，node 中间层接收到这个请求的文章 id ，知道这个需要是服务端渲染的页面，再转发到后端，后端返回渲染好的 html 文件给 node 中间层， node 中间层再将渲染好的 html 文件返回给浏览器，浏览器再将其显示出来，这样就实现了服务端的渲染。
