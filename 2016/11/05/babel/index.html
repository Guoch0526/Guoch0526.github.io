    • babel学习

    2016-11-05
    • Study notes

    由于浏览器对ES2015(es6)新的语法的支持并不是很理想，所以如果我们要使用es6来进行开发的话会带来一些问题。目前的解决方案是使用es6编写代码，最后将es6代码转为es5代码。

    babel 是一个将ES6甚至ES2017转化为ES5代码的一个工具包。在前端自动化构建中可以使用这个工具包进行代码的转换
    注意：babel只能将普通的变量声明等方式，函数的命名方式转换，并没有办法将不支持的函数方法转换。所以需要在引入语法包,来支持新的扩展函数.

    1.引入babel包

    2.配置.babelrc文件

    新建 .babelrc 文件。

    • 方法1： window右键新建文本文件 输入 .babelrc. 确认之后就可以了。
    • 方法2: 直接使用webStorm 创建
    • 方法3： 使用git bash 创建
      • touch .babelrc
      • echo .babelrc '{ "preset":["es2015"] }'

    • .babelrc 文件内容

    3.下载语言转换包。

    //如果还没有npm init初始化的需要先初始化。
//下载在当前文件夹，并且保存在pageage.json中的开发依赖项

npm install babel-preset-es2015 --save-dev

    4.babel 命令

    babel 要转换的文件路径  -o  要保存的文件路径  （包括文件名.js）
    • 例子
      • babel index.js -o index1.js

    注意：

    babel只能将es6语法转换为es5，但是并不能将es6新扩展的方法函数给转换。如promies异步函数等。
在ie中，并不能很好的支持es6.所以如果我们需要在浏览器中使用es6的新函数，那么就需要引入es6的扩展包。
在babel中提供了一个polyfill.js文件来当作垫片。需要在html文件中引入这个js文件来支持es6扩展的函数方法

    5.下载 babel–polyfill垫片；

    • 需要在html中引入这个polyfill这个文件就好了。
    • 在需要扩展es6代码的文件中，使用 require('babel-polyfill'); 来引入这个扩展包。

    扩展

    引入es6新扩展的函数包，是一个比较耗性能的问题，必定会产生新的问题。
